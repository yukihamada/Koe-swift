## P1. 田中 健太郎（34・Mac native dev）

### 1. プロフィール
- **仮名**: 田中 健太郎（たなか けんたろう）
- **年齢**: 34
- **居住**: 東京都 渋谷区
- **家族構成**: 妻（同い年、リモート）+ 子なし
- **職業**: SaaS スタートアップ Tech Lead（iOS + macOS native）
- **年収**: 900 万円
- **デバイス**: M4 Max MacBook Pro 14"、iPhone 16 Pro、HomePod mini

### 2. 状況とジョブ
> "Slack の返信を音声で済ませて、コードに戻りたい。打鍵時間が日課の足を引っぱってる。"

普段は Ghostty + Xcode + Safari 多用。日中チャットに割く時間が長すぎて、エディタにいる時間が削られている。Koe を Slack/GitHub PR の返信高速化に使いたい。

### 3. ペイン
- 「Ghostty で `⌥⌘V` 押したら **既存のクリップボード貼付** とコンフリクトする」
- 「英単語が混ざる発話を強制日本語認識されると技術用語が壊滅する（"async/await" が「あしんく あう」に）」
- 「Settings 開いてもどの tab に何があるか直感的じゃない。`Fn` 設定どこ？」
- 「録音中の音量 5% は会議中だと音楽が止まったように聞こえる。配偶者が後ろで電話してたら気まずい」
- 「Apple Speech vs whisper.cpp の挙動差がドキュメント化されてないから消去法で詰める」

### 4. ゲイン
- 「日本語で書いたあと英語の専門用語を交ぜても両方正しく出る → "Auto Detect" が信用できる」
- 「`Fn` 単発で recording トグルできる → ホームポジション崩さない」
- 「per-app profile で Ghostty 専用 prompt が effective」
- 「2,000 件くらいの履歴を search できる」

### 5. 情報接触経路
GitHub Trending → README → Releases ページの changelog → DMG ダウンロード → 起動 → Settings → ⌥⌘V でいきなり試す

### 6. プロダクトに来る動機（一人称の問い）
1. 「これ本当に Apple Silicon ネイティブ？ Rosetta 2 じゃないよね？」
2. 「`async/await` を日本語混じりで言ったら正しくスペル出る？」
3. 「設定多すぎ → デフォルトでとりあえず使える？」
4. 「ホットキーが他アプリと被ったら教えてくれる？」
5. 「履歴って暗号化されてる？平文 .log だと盗まれそう」

### 7. コンバージョン経路
- **第一**: ⌥⌘V → 1分以内に「使える」と判断 → Slack 用に常駐
- **第二**: per-app profile を 5 個カスタマイズ → daily driver 化

### KPI
- 1日あたり録音回数 ≥ 20 回
- 録音 → ペースト までの median latency < 2.5s

### 6. プロダクトに来る動機
1. 「録音データ、私の Mac から 1 バイトも出ない？」

5. 「Big Sur (macOS 11) のサブ機でも動く？」

### 7. コンバージョン経路

### 3. ペイン
- 「Settings のラベルが読めない（VoiceOver 対応 90% 程度の所感）」
- 「Fn キー単発タップ、説明が "tap_toggle" "hold_ptt" だけ。**動作のプレビュー** か **デモ動画** が欲しい」

- 「Carbon hotkey 4 種類（メイン + 翻訳 + ⌃K + ⌥⌘M + ⌃⌥R）全部 register 失敗時の error が `klog` だけで UI に出ない」
- 「腱鞘炎なので **マウスクリック数も最小化** したい。Settings → Automation tab までクリック 4 回はキツい」

### 4. ゲイン
- 「Fn 単発タップで録音 → Wake Word で 再起動 → 全てキー操作なし」
- 「VoiceOver が全画面正しく読む（プレビュー → 試聴で確認できる）」
- 「Settings 内 search ボックスで "fn" と入れたら直接該当箇所に飛ぶ」
- 「権限不足時に **具体的にどこを開けばいいか** のステップガイド付き alert」

### 5. 情報接触経路
社内 アクセシビリティ Slack → 同業の老 SRE 紹介 → 公式 README → DMG

### 6. プロダクトに来る動機
1. 「タイピング 100% 代替できる？コマンド系の voice ショートカット使える？」
2. 「Fn は MacBook の ESC 兼用 globe key も検知する？」
3. 「Wake Word の誤検知率は？私の環境（オフィス + テレビ）でテストできる？」
4. 「VoiceOver で全 Settings 操作可能？」
5. 「サポートに **日本語で**問い合わせられる？」

### 7. コンバージョン経路

### KPI

- Fn key + Wake Word 利用率 > 80%
